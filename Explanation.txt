OVERVIEW
--------
WebNews is a Next.js 15 application that combines e-commerce functionality with a content management system. It serves as a platform for selling digital products while also managing web news content with YouTube video integration.

CORE FUNCTIONALITY
------------------

1. E-commerce System
   - Products can be created with name, description, price, downloadable file, and image
   - Customers browse products on the public-facing storefront
   - Stripe handles payment processing
   - After purchase, customers receive email with download links
   - Download links are time-limited (24-hour expiration) via DownloadVerification tokens

2. Content Management (WebNews)
   - Hierarchical content structure: Page → Type → Column
   - Supports both paper-based content and YouTube videos
   - YouTube integration fetches video metadata (title, thumbnail, duration) automatically
   - Bulk import via CSV file upload with PapaParse processing
   - Core metadata split across 8 fields (core1-core8) for flexible content categorization

3. Admin Dashboard
   - Protected area requiring authentication
   - Statistics overview: total sales, customer count, product count, news items
   - CRUD operations for products, users, orders, and web news
   - Product management includes file/image uploads
   - Order management with deletion capability
   - User management with order history

USER FLOWS
----------

Customer Flow:
1. Browse products on homepage
2. Select product to view details
3. Proceed to Stripe checkout
4. Complete payment
5. Receive purchase confirmation email with download link
6. Access download (valid for 24 hours)
7. Can request order history via email at any time

Admin Flow:
1. Login with credentials (username/password validated against env vars)
2. View dashboard statistics
3. Manage products (add/edit/delete with file uploads)
4. Manage web news content (manual entry or YouTube URL)
5. View and manage customer orders
6. Delete users if needed

TECHNICAL ARCHITECTURE
----------------------

Frontend:
- Next.js App Router with React Server Components
- Tailwind CSS for styling
- shadcn/ui component library (built on Radix UI)
- Progressive enhancement with useFormState/useFormStatus
- Suspense boundaries with skeleton loaders

Backend:
- Server Actions for form handling and mutations
- Zod schema validation with field-level error returns
- SQLite database via Prisma ORM
- NextAuth for session management
- Hybrid caching strategy (React cache + Next.js unstable_cache)

Data Flow:
- Server components fetch data directly from database
- Mutations go through server actions
- Cache invalidation via revalidatePath() after changes
- File uploads stored on filesystem (products/ and public/images/)

External Integrations:
- Stripe: Payment processing and webhooks
- Resend: Transactional email delivery
- React Email: Email template rendering
- YouTube Data API: Video metadata fetching
- Google APIs: OAuth and playlist access

DATABASE SCHEMA
---------------

Product: Stores sellable items with pricing, files, and images
User: Customer accounts identified by email
Order: Links users to purchased products with price paid
DownloadVerification: Time-limited tokens for secure file downloads
WebNews: Content items with hierarchical organization and metadata

SECURITY CONSIDERATIONS
-----------------------
- Admin authentication via hashed passwords (SHA-256)
- Session management through NextAuth with httpOnly cookies
- Download links expire after 24 hours
- File paths use UUIDs to prevent enumeration
- Environment variables store sensitive credentials

DEVELOPMENT SETUP
-----------------
- Development server runs on port 3005
- Email preview available on port 3011
- SQLite database for easy local development
- Custom logging system writes to /logging/logging.json
- Dutch (nl-NL) locale used for currency formatting
