WebNews - Improvements and Implementation Plan
==============================================

This document outlines identified issues and a prioritized plan for addressing them.

The codebase works fine for exploring Next.js patterns and building out features.                                                                           
  If you ever decide to deploy it publicly, the key things to address first would be: 
    - Fix isValidPassword.ts (currently bypassed)
    - Add admin route protection middleware                                                                                                                                                                                                                       
  The other items in the plan (performance, code quality, type safety) are more about code maintainability and could be tackled incrementally as you continue developing.

================================================================================
PART 1: IDENTIFIED IMPROVEMENTS
================================================================================

------------------------------------------------------------------------------
CRITICAL SECURITY ISSUES
------------------------------------------------------------------------------

1. PASSWORD VALIDATION DISABLED
   File: src/lib/isValidPassword.ts
   Issue: Function unconditionally returns true, bypassing all authentication.
   The entire validation logic is commented out.

2. WEAK PASSWORD HASHING (SHA-256)
   Files: src/app/api/auth/[...nextauth]/route.ts
          src/app/api/admin/login/route.ts
   Issue: SHA-256 is not suitable for password hashing:
   - Not salted (vulnerable to rainbow tables)
   - Too fast (vulnerable to brute force)
   - Not designed for password storage
   Recommendation: Use bcrypt, scrypt, or argon2.

3. API KEYS EXPOSED IN URL QUERY PARAMETERS
   Files: src/app/api/get-playlists/route.ts
          src/app/admin/_actions/webnews.ts
          src/app/admin/youtube/[id]/videolist/page.tsx
   Issue: Access tokens and API keys visible in browser history, server logs,
   and referrer headers.
   Recommendation: Pass tokens in Authorization headers.

4. NO CSRF PROTECTION
   File: src/app/api/admin/login/route.ts
   Issue: POST endpoint lacks CSRF token validation.

5. INSECURE SESSION COOKIE
   File: src/app/api/admin/login/route.ts
   Issue: Cookie value is just 'true' - predictable and insecure.
   Missing SameSite attribute.

6. NO RATE LIMITING ON LOGIN
   File: src/app/api/admin/login/route.ts
   Issue: Unlimited login attempts allowed - vulnerable to brute force.

7. NO ADMIN ROUTE PROTECTION
   File: src/app/admin/layout.tsx
   Issue: Admin routes have no middleware checking authentication.
   Anyone can access /admin/* directly.

------------------------------------------------------------------------------
CODE QUALITY ISSUES
------------------------------------------------------------------------------

8. EXCESSIVE USE OF 'any' TYPE
   Found: ~90 occurrences across 36 files
   Issue: Breaks TypeScript's type safety guarantees.

9. DELETE BEFORE VALIDATE PATTERN
   Files: src/app/admin/_actions/products.ts (deleteProduct)
          src/app/admin/_actions/users.ts
          src/app/admin/_actions/orders.ts
   Issue: Records deleted before checking if they exist.
   File cleanup happens after deletion - if files don't exist, database
   is already modified.

10. HARDCODED ARRAY INDICES
    File: src/app/admin/page.tsx (WebNewsCard component)
    Issue: Accesses body[0] through body[27] directly.
    Will crash if array has fewer than 28 items.
    Duplicate key: body[20] appears twice.

11. INCOMPLETE FUNCTION
    File: src/app/admin/_actions/webnews.ts (deleteSectionsWebNews)
    Issue: Entire function body is commented out but function is exported.
    Silently does nothing when called.

12. RACE CONDITIONS IN FILE UPLOAD
    File: src/app/api/upload/route.ts
    Issue: Uses wait(100) delays to work around timing issues.
    Sequential database inserts are slow.

13. INCONSISTENT FILE PATH HANDLING
    File: src/app/admin/_actions/products.ts
    Issue: Product files go to /products/, images go to public/images/.
    Different access patterns, hard to maintain.

14. MALFORMED LOGGING
    File: src/lib/logging.tsx
    Issue: Generates invalid JSON (starts with comma).
    Silently swallows all logging errors.

15. ASYNC/AWAIT ANTI-PATTERN
    File: src/actions/orders.tsx
    Issue: Using await inside .map() instead of Promise.all().

------------------------------------------------------------------------------
PERFORMANCE ISSUES
------------------------------------------------------------------------------

16. N+1 QUERY PROBLEM
    File: src/app/admin/page.tsx (getWebNewsData)
    Issue: Loads all records, then queries again per page, then per type.
    For 20 pages x 5 types = ~110+ queries.
    Should use single aggregation query.

17. ARTIFICIAL DELAYS
    File: src/app/admin/page.tsx (getSalesData)
    Issue: Contains wait(2000) - delays dashboard by 2 seconds.

18. NO PAGINATION
    File: src/app/admin/orders/page.tsx
    Issue: Loads ALL orders into memory. Will cause memory bloat and
    slow rendering with thousands of orders.

19. NO DOWNLOAD VERIFICATION CLEANUP
    File: src/app/webhooks/stripe/route.tsx
    Issue: Creates records with expiration but never cleans them up.
    Database grows indefinitely with dead records.

------------------------------------------------------------------------------
MISSING FEATURES
------------------------------------------------------------------------------

20. NO EMAIL RETRY LOGIC
    File: src/app/webhooks/stripe/route.tsx
    Issue: If email fails, order exists but customer never gets receipt.

21. NO TRANSACTION SUPPORT FOR FILE OPERATIONS
    File: src/app/admin/_actions/products.ts
    Issue: If file upload fails mid-way, database and filesystem diverge.

22. ENVIRONMENT VARIABLES NOT VALIDATED
    Issue: No startup validation for required env vars.
    App crashes at runtime instead of startup.

23. MIXED AUTHENTICATION IMPLEMENTATIONS
    Files: src/auth.ts (Google provider)
           src/app/api/auth/[...nextauth]/route.ts (Credentials)
           src/app/api/admin/login/route.ts (Custom)
    Issue: Three different auth approaches, unclear which is active.
